# QA frontend

## Revisión obligatoria

- Mobile 360/390/430.
- Desktop.
- Sin overflow horizontal.
- Contraste AA.
- Foco visible.
- Datos sensibles correctos.
- No mostrar datos personales en token inexistente.
- No usar clichés IA.

## Pantalla pública

- estado visible sin scroll excesivo;
- fechas presentes claras;
- QR/link comprensibles.

## Admin

- acciones principales visibles;
- desktop eficiente;
- mobile usable;
- acciones críticas separadas.
