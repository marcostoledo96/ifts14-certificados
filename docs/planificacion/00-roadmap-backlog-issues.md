# Roadmap, backlog e issues

## Roadmap

Ver `docs/04-planificacion-github-roadmap.md`.

## Backlog resumido

- DOC: documentación base.
- SPEC: specs SDD.
- FE: pantallas públicas/admin.
- BE: API y casos de uso.
- PDF: PDF/QR.
- EMAIL: envío/reenvío.
- QA: pruebas y demo.
- DEP: deploy.
- OP: operación/handoff.

## Reglas de issues

Cada issue debe incluir:

- objetivo;
- archivos esperados;
- criterios de aceptación;
- checklist QA;
- docs a actualizar.
