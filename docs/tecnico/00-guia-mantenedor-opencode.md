# Guía técnica para mantenedor y OpenCode

## Flujo recomendado

1. Leer `AGENTS.md`.
2. Leer índice.
3. Abrir spec específica.
4. Revisar fixture/contrato.
5. Planificar con agente `plan`.
6. Implementar con agente de build.
7. Probar.
8. Actualizar docs.

## Reglas

- OpenCode no debe commitear automáticamente.
- No leer archivos largos sin necesidad.
- No modificar contratos sin actualizar fixtures.
- No asumir stack final.
