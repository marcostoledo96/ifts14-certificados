# AGENTS.md — Fixtures

## Propósito

Los fixtures son datos de ejemplo que funcionan como contrato temporal entre frontend y backend.

## Reglas

- No cambiar nombres de campos sin actualizar contratos.
- Cada fixture debe representar un caso real.
- No usar datos personales reales.
- Mantener JSON válido.
- Si cambia un fixture, revisar specs y frontend/backend.
